composer - менеджер зависимостей для php

установка composer:
sudo apt update

установка php:
sudo apt install php

проверка установки:
php -v

установка установщика:
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"

локальная установка:
php composer-setup.php

глобальная установка(это позволит диспетчеру зависимостей запускаться в любом месте системы без указания полного пути):
php composer-setup.php --install-dir=/usr/local/bin --filename=composer

удаление инсталлятора:
php -r "unlink('composer-setup.php');"

запуск:
composer

обновление пакетов(!!!ОПАСНО ИСПОЛЬЗОВАТЬ БЕЗ ТЕСТИРОВАНИЯ): composer update

РАБОТА С COMPOSER:

пр-р: необходимо установить пакет для работы с арифметикой в папку с проектом progect

1)найти пайти нужный пакет на сайте https://packagist.org у нас это brick/math
2)переходим в папку проекта
/opt/lampp/htdocs/progect

Теперь нам нужно установить наш пакет brick/math чтобы использовать его в PHP-скрипте.
composer require brick/math

теперь проверим файлы командой ls -l:

итого 12
-rw-rw-r-- 1 lada lada 59 дек 4 17:16 composer.json
-rw-rw-r-- 1 lada lada 2575 дек 4 17:15 composer.lock
drwxrwxr-x 4 lada lada 4096 дек 4 17:16 vendor

vendor - каталог, где хранятся зависимости проекта.
composer.lock - более подробная информация об установленных пакетах чем composer.json.(автор, лицензии, список зависимотей для корект. работы и т.д.)
composer.json - файл где хранятся все установленные пакеты и их версии

пр-р:

{
"require": {
"brick/math": "^0.12.1"
}
}

чтобы пользоваться пакетами в php скриптах необходимо прописать в начале файл автозагрузчик:
require __DIR__ . '/vendor/autoload.php';

УДАЛЕНИЕ ПАКЕТОВ:
composer remove brick/math

файлы и каталоги останутся, но composer.json:
{
}
